# MazeTD 数值设计文档 V2.0

## 一、设计理念

### 1.1 核心原则
- **易上手难精通**：前3关教学引导，后9关逐步挑战
- **策略多样性**：每种塔有独特定位，没有绝对最优解
- **资源约束**：金币有限，迫使玩家做选择
- **风险回报**：高风险高收益的设计

### 1.2 参考标杆
- 保卫萝卜：关卡渐进、新手友好
- 皇室战争：资源博弈、时机把控
- Kingdom Rush：塔种克制、策略深度

## 三、关卡系统

### 3.1 章节结构

**第一章：入门篇（关卡1-3）**
- 目标：学习基础操作
- 塔解锁：墙壁、弓箭塔、炮塔
- 敌人：普通怪、快速怪
- 难度：极简，保证通关

**第二章：成长篇（关卡4-6）**
- 目标：掌握策略组合
- 塔解锁：冰冻塔、毒塔、陷阱墙
- 敌人：引入坦克怪、侦察兵
- 难度：中等，需要思考

**第三章：挑战篇（关卡7-9）**
- 目标：综合策略运用
- 塔解锁：雷电塔、阻拦塔
- 敌人：引入破坏者、变形怪
- 难度：较高，考验策略

**第四章：精英篇（关卡10-12）**
- 目标：极限挑战
- 塔解锁：传送塔、全部塔
- 敌人：混合出怪、BOSS战
- 难度：极高，需要完美策略

### 3.3 波次生成规则

```
波次怪物数量公式：
baseCount = 3 + waveIndex * 2
maxCount = min(baseCount, 20)

敌人类型权重（根据关卡配置）：
- 普通：50%
- 快速：25%
- 坦克：15%
- 特殊：10%（侦察/破坏者/变形怪）

BOSS波次：
- 最后一波必定出BOSS
- BOSS血量 = 基础血量 * (1 + 关卡Index * 0.5)
- BOSS同时召唤小怪
```

## 四、玩法逻辑

### 4.1 迷宫建造
- 玩家可自由放置墙壁改变怪物路径
- 墙壁有血量，可被破坏者攻击
- 路径不能完全堵死（必须有通路）

### 4.2 塔放置
- 塔只能放在空白格子
- 阻拦塔同时作为障碍物
- 传送塔需要配合迷宫设计

### 4.3 战斗流程
1. 准备阶段：建造/升级塔
2. 开战阶段：波次自动进行
3. 波次间隔：短暂休息，可调整布局
4. 通关/失败条件

### 4.4 AI行为
- 侦察兵：记录玩家火力点，后续波次避开
- 破坏者：主动攻击墙壁开辟新路径
- 变形怪：每5秒切换属性（火/冰/毒）

## 五、平衡参数

### 5.1 伤害公式
```
实际伤害 = 基础伤害 * (1 + 连击加成) * 暴击倍率
连击加成 = min(连续命中次数 * 0.05, 0.5)
暴击倍率 = 1.5（5%概率）
```

### 5.2 经济平衡
```
每关预期收益：
- 击杀收益 ≈ 初始金币 * 1.5
- 通关奖励 ≈ 剩余血量 * 5

塔投资回报：
- 弓箭塔：3-4波回本
- 炮塔：4-5波回本
- 高级塔：5-6波回本
```

### 5.3 难度曲线
```
前3关：玩家胜率 95%+
4-6关：玩家胜率 80%+
7-9关：玩家胜率 60%+
10-12关：玩家胜率 40%+
```
